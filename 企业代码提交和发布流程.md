**目的**

带大家了解企业团队协作一个项目的规范流程



## 企业中如何管理项目代码

GitHub 代码开源托管平台

**企业的代码是以项目的形式托管在平台上的**

### 怎么协作开发？怎么多人给同一个项目提交

vcs工具 （版本控制系统），**Git**:通过命令行的方式提交、发布、拉取代码、提高协作效率。

或者 svn



## 1.拉取代码

```bash
git clone <输入网址> 
```





## 2.本地开发

直接提交代码，是直接影响了主分支

为了保证线上项目的稳定，我们要保护分类，不能直接多人在主分支去开发

每个程序员在开发时，要有自己的分支

**分支规范**：

1.每个开发者一个独立分支

2.每一个需求一个分支，粒度更细



创建和切换分支

```bash
git checkout -b <name>
```

查看当前分支和状态

```bash
git status
```

本地写代码  





## 3.提交代码

添加本次修改

```bash
git add .
```

遵循提交规范：

```bash
add xxx
update xxx
rm xxx
```

代码提交

```bash
git commit -m '提交信息'
```



## 4.提交到远程

先和远程分支关联

本地仓库==》远程仓库

本地有一个分支==》远程的一个分支（本地仓库和远程仓库的关联）



推送

```bash
git push
```

只会影响
